# Event Hubs

- Event Hubs is a big data streaming platform and event ingestion service
- Event Hubs is basically a managed Kafka implementation
- It can receive millions of events per second

## Event Hubs Components

- Event Producers:
    - Components that are generating the events
    - Events can be generated by anyone with a client or with a HTTP client
    - Events producers only need to utilize simple API connections
- Partitions:
    - Partitions represent single event streams
    - We can think of them as a single queue
    - They guaratee the order of the messages
    - Partitions have limited availability, there is no way to make a partition HA
    - Better to spread messages across partitions to improve availability => the order of the messages will not be guaranteed
    - We can have max 32 partition on a single Event Hubs deployment
- Consumer Group:
    - It is a logical group of receivers, belonging to the same application
    - Example:
        - Receivers for processing telemetry
        - Receivers for storing telemetry
    - Event receiving is done via AMQP protocol

## Availability

- SLA:
    - Basic and Standard tiers: 99.95%
    - Dedicated: 99.99%

## Throughput Units (TU)

- Throughput is measured in Throughput Units
- 1 TU = 
    - Ingress: 1 MB / sec or 1000 events / sec
    - Egress: 2 MB / sec or 4096 events / sec
- TU as prepurchased, billed by the hour

## Pricing

- It is based on:
    - Tier
    - Ingress
    - Throughput Units (TU)
