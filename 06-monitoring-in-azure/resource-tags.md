# Resouce Tags

- Resouces are organized into Resource Groups
- Sometimes we want more information about resources such as:
    - Which environment it belongs? (test, prod)
    - To which app it blongs?
    - Who is responsible for the resource?
- Tags help organize resources using name-value pairs
- Examples:
    - Environment: Test
    - App: appname
    - Group: teamname
- Tags can be set during creaton of the resource or anytime after that
- Tags are useful for:
    - Resource querying
    - Cost analysis
